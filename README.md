R/M Base — это веб-приложение на Flask для управления информацией о рок- и метал-группах, музыкантах, альбомах, песнях и гитарных разборах. Приложение предоставляет как удобный веб-интерфейс, так и RESTful API.

## Возможности

- Просмотр списков групп, музыкантов, альбомов и песен
- Детальные страницы для каждой группы, музыканта и альбома
- Аутентификация пользователей (регистрация / вход)
- Админ-панель для добавления, редактирования и удаления контента
- Загрузка обложек альбомов и фото музыкантов
- Загрузка MP3-файлов песен
- Добавление ссылок на YouTube-разборы для песен
- REST API для групп, музыкантов и пользователей
- База данных SQLite с SQLAlchemy ORM

## Используемые технологии

- **Backend**: Flask, Flask-RESTful, Flask-Login, Flask-WTF
- **База данных**: SQLite, SQLAlchemy
- **Frontend**: HTML, Bootstrap 4, custom CSS
- **Хранение файлов**: Локальные статические файлы (изображения, аудио)

## Схема базы данных

База данных `music.db` состоит из следующих таблиц:

### albums

| Колонка       | Тип     | Описание                       |
|---------------|---------|--------------------------------|
| id            | INTEGER | Первичный ключ, автоинкремент  |
| created_date  | STRING  | Дата выпуска альбома           |
| name          | VARCHAR | Название альбома               |
| group_id      | INTEGER | Внешний ключ к `groups.id`     |

### groups

| Колонка       | Тип      | Описание                       |
|---------------|----------|--------------------------------|
| id            | INTEGER  | Первичный ключ, автоинкремент  |
| name          | VARCHAR  | Название группы                |
| genre         | STRING   | Жанр                           |
| created_date  | DATETIME | Дата основания                 |
| closed_date   | DATETIME | Дата распада (если есть)       |
| short_bio     | TEXT     | Краткая биография              |

### musicians

| Колонка       | Тип      | Описание                       |
|---------------|----------|--------------------------------|
| id            | INTEGER  | Первичный ключ, автоинкремент  |
| name          | VARCHAR  | Имя                            |
| surname       | VARCHAR  | Фамилия                        |
| birth_date    | DATETIME | Дата рождения                  |
| death_date    | DATETIME | Дата смерти (если есть)        |
| group_id      | INTEGER  | Внешний ключ к `groups.id`     |
| short_bio     | TEXT     | Краткая биография              |

### songs

| Колонка       | Тип     | Описание                              |
|---------------|---------|---------------------------------------|
| id            | INTEGER | Первичный ключ, автоинкремент         |
| name          | VARCHAR | Название песни                        |
| album_id      | INTEGER | Внешний ключ к `albums.id`            |
| tutor         | VARCHAR | Ссылка на YouTube-разбор (или "None") |

### users

| Колонка          | Тип      | Описание                            |
|------------------|----------|-------------------------------------|
| id               | INTEGER  | Первичный ключ, автоинкремент, уникальный |
| name             | STRING   | Имя пользователя                    |
| email            | STRING   | Уникальный email                    |
| hashed_password  | STRING   | Хеш пароля                          |
| created_date     | DATETIME | Дата регистрации                    |
| is_admin         | BOOLEAN  | Флаг администратора (по умолчанию 0)|

## Установка и запуск

1. **Клонируйте репозиторий**
   ```
   git clone <url-репозитория>
   cd R/M-Base
   ```

2. **Создайте виртуальное окружение (рекомендуется)**
   ```
   python -m venv venv
   source venv/bin/activate   # Linux/macOS
   venv\Scripts\activate      # Windows
   ```

3. **Установите зависимости**
   
   Все необходимые зависимости перечислены в файле `requirements.txt`. Установите их одной командой:
   
   ```
   pip install -r requirements.txt
   ```

4. **Инициализируйте базу данных**
   
   База данных создастся автоматически при первом запуске приложения. Убедитесь, что директория `db/` существует.

5. **Запустите приложение**
   ```
   python main.py
   ```
   
   Приложение будет доступно по адресу `http://127.0.0.1:8000`

## API Endpoints

| Метод   | URL                         | Описание                       |
|---------|-----------------------------|--------------------------------|
| GET     | `/api/bands`                | Получить список всех групп     |
| GET     | `/api/bands/<id>`           | Получить конкретную группу     |
| POST    | `/api/bands`                | Добавить новую группу          |
| DELETE  | `/api/bands/<id>`           | Удалить группу                 |
| GET     | `/api/musicians`            | Получить список всех музыкантов|
| GET     | `/api/musicians/<id>`       | Получить конкретного музыканта |
| POST    | `/api/musicians`            | Добавить нового музыканта      |
| DELETE  | `/api/musicians/<id>`       | Удалить музыканта              |
| GET     | `/api/users`                | Получить список всех пользователей |
| GET     | `/api/users/<id>`           | Получить конкретного пользователя |
| POST    | `/api/users`                | Добавить нового пользователя   |
| DELETE  | `/api/users/<id>`           | Удалить пользователя           |

> **Примечание**: Для POST и DELETE запросов через API могут потребоваться права администратора.

## Примеры использования

### Веб-интерфейс

- **Главная страница**: `/`
- **Список групп**: `/groups`
- **Список музыкантов**: `/musicians`
- **Разборы** (для авторизованных пользователей): `/tutorials`
- **Действия администратора**: видны только пользователям с `is_admin=True`

### Примеры API-запросов

```
# Получить все группы
curl http://localhost:8000/api/bands

# Добавить новую группу
curl -X POST http://localhost:8000/api/bands \
  -H "Content-Type: application/json" \
  -d '{"name":"Новая группа","genre":"Метал","created_date":"2024-01-01"}'
```

## Обработка ошибок

Приложение возвращает соответствующие страницы ошибок для:

- `400` – Bad Request (Неверный запрос)
- `404` – Not Found (Не найдено)
- `500` – Internal Server Error (Внутренняя ошибка сервера)

API-ошибки возвращают JSON с полем `error`.
